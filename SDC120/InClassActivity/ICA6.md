## Objective
Learn to think in state machines, not if-statements.

## Activity
As a class, students analyze a real-world system (examples):
- ATM machine
- Online order lifecycle
- Game character health

Students must:
- Identify all possible states
- Identify valid transitions
- Identify invalid transitions
- Diagram the state flow

## Deliverable
- State diagram (paper, whiteboard, or digital)
- List of blocked transitions
